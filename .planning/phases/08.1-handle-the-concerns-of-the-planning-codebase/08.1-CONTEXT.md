<specifics>
## Specific Ideas

- User describes the core context system problem as: "the agent uses caching but lacks either the understanding or tools to process multi-tasks written in human language and parse it — even after a few successful sub-phases there is sub-task amnesia and lack of result chunk to sub-task affiliation." The system needs better mapping between indicators, results, and tooling to be stable and process-based. This frames the Phase 8.2 redesign spec.
- The ContextManager redesign (Phase 8.2) should address: mission-boundary-aware eviction (not just a fixed message window), explicit and testable injection policy, and clear result-to-subtask affiliation in the state schema.

</specifics>
