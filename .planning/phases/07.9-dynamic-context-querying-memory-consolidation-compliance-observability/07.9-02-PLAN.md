  <behavior>
    - Test: _cosine_similarity([1,0,0], [1,0,0]) == 1.0
    - Test: _cosine_similarity([1,0,0], [0,1,0]) == 0.0
    - Test: _cosine_similarity([0,0,0], [1,0,0]) == 0.0 (zero vector edge case)
    - Test: _cluster_by_similarity with 4 items where 2 pairs have sim >= 0.85 -> returns 2 clusters
    - Test: _cluster_by_similarity with all items similar -> returns 1 cluster
    - Test: _cluster_by_similarity with no items similar -> returns N singleton clusters
    - Test: _merge_cluster_summary combines goals and tools_used from cluster members
    - Test: consolidate_memories with pool=None raises or returns early gracefully
  </behavior>
  <action>
    1. Create `src/agentic_workflows/storage/memory_consolidation.py`:
       - `_cosine_similarity(a: list[float], b: list[float]) -> float` — pure Python dot product / (mag_a * mag_b), return 0.0 for zero vectors
       - `_cluster_by_similarity(items: list[dict], threshold: float = 0.85) -> list[list[dict]]` — greedy single-linkage clustering. Each item dict has "id", "embedding", "goal", "summary", "tools_used", etc. Items are grouped if any pair in cluster has similarity >= threshold.
       - `_merge_cluster_summary(cluster: list[dict]) -> str` — combine goals: "Consolidated N missions: {goal1}; {goal2}; ..." truncated to 500 chars. Merge tools_used as union set.
       - `_row_to_item(row) -> dict` — convert DB row tuple to dict with id, run_id, mission_id, goal, summary, tools_used (parsed from text[]), embedding (parsed from vector string)
       - `consolidate_memories(pool, embedding_provider=None, age_days=7, similarity_threshold=0.85) -> dict[str, Any]`:
         - Early return `{"clusters": 0, "consolidated": 0, "kept": 0}` if pool is None
         - `with pool.connection() as conn:` fetch old completed missions with embeddings
         - SQL: `SELECT id, run_id, mission_id, goal, summary, tools_used, embedding FROM mission_contexts WHERE status = 'completed' AND created_at < NOW() - INTERVAL '%s days'` with age_days param
         - Skip rows with NULL embedding (can't cluster without embeddings)
         - Cluster, then for each cluster with 2+ items: DELETE originals by id, INSERT consolidated record using first item as base with merged summary. If embedding_provider available, re-embed merged summary; else average cluster embeddings.
         - Return counts dict
    2. Create `tests/unit/test_memory_consolidation.py` testing _cosine_similarity, _cluster_by_similarity, _merge_cluster_summary. Use deterministic 3-dim vectors for clarity (not 384-dim). Test consolidate_memories with pool=None returns early.
  </action>
