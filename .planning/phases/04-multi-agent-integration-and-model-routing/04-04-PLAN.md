<objective>
Fix the mission attribution regression introduced in Plan 04-01. When `_route_to_specialist()` replaced `_execute_action()` with `self._executor_subgraph.invoke()`, the call to `_record_mission_tool_event()` was lost. This leaves `mission_reports[*].used_tools` empty for all subgraph-routed tool calls, causing the MissionAuditor `required_tools_missing` check to fire for every mission with a required_tools contract — 26 integration tests fail as a result.

Purpose: Restore full MAGT-05 and MAGT-06 compliance by closing the mission attribution gap in the subgraph routing path.
Output: `_route_to_specialist()` in graph.py with `_record_mission_tool_event()` + `tool_call_counts` increment added after both executor and evaluator copy-back loops; all 26 integration regressions fixed.
</objective>
