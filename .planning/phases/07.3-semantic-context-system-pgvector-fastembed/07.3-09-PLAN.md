    ## Sections to include

    **1. Phase Overview**
    - What this phase adds (persistent cross-run mission context with hybrid retrieval)
    - What it does NOT do (graph.py decomposition → 7.4, Send() → 7.5, LLM summaries → post-7.3)

    **2. Why Hybrid Retrieval?**
    - Pure vector search fails for exact matches (hash should short-circuit, not score)
    - BM25 catches keyword matches that confuse embeddings
    - The industry pattern (Zep, Haystack, Hindsight) proves deterministic+semantic cascade
    - +15-30% recall improvement over single-list approaches

    **3. The 5-Layer Cascade (L0-L4)**
    - L0: SHA-256 exact hash — why this is first (fastest, zero Postgres work)
    - L1: Tool bitmask — structural matching, 37 tools fit in 64-bit int
    - L2: tsvector BM25 — native Postgres, indexed, deterministic
    - L3: BIT(384) binary Hamming — merged with L4 pipeline for simplicity
    - L4: vector(384) HNSW cosine — Reciprocal Rank Fusion with L2
    - Short-circuit semantics: L0 and L1 can return early

    **4. Embedding Stack**
    - fastembed vs sentence-transformers decision (no PyTorch, Qdrant-maintained)
    - BAAI/bge-small-en-v1.5: 384-dim, 24MB, top MTEB for its size class
    - MockEmbeddingProvider: SHA-256 seed → deterministic unit-norm vector
    - EMBEDDING_PROVIDER env var pattern
    - Code example: MockEmbeddingProvider.embed_sync()

    **5. Reciprocal Rank Fusion**
    - What it is (Cormack et al. 2009)
    - The formula: 1/(k + rank + 1) where k=60
    - Why k=60 (standard, zero tuning, production-proven)
    - Why no tunable weights (they don't help vs RRF in practice)
    - Code example: reciprocal_rank_fusion()

    **6. BIT(384) Binary Quantization**
    - Why: 32x smaller than float32, 10x faster Hamming vs cosine, 0.3% recall loss
    - How: sign bit of each float dimension → '0' or '1' string for Postgres BIT type
    - Code example: _float_vec_to_bit_string()
    - The pitfall: must pass as plain string to psycopg, not bytes

    **7. HNSW vs IVFFlat**
    - HNSW: no training phase, better accuracy for small-to-medium collections (<1M rows)
    - IVFFlat: requires training, accuracy depends on nlist parameter
    - Decision: HNSW for this project

    **8. ContextManager Integration**
    - Optional params pattern (backward compat — zero-arg callers unaffected)
    - _persist_mission_context(): called at end of on_mission_complete(), sync, no-op if store=None
    - build_planner_context_injection(): adds [Cross-run] Similar: hits, 1500-char cap
    - Graceful degradation: all exceptions caught with warning log, never crash the graph

    **9. Tool Bitmask Registry**
    - TOOL_BITS dict: 37 tools, bit 0-36, fits in BIGINT
    - Append-only: new tools get higher bit positions
    - encode_tool_pattern() and L1 query pattern

    **10. Running the Stress Test**
    ```bash

    docker compose -f docker/docker-compose.stress.yml up --build

    # In another terminal, run the stress test

    ```
