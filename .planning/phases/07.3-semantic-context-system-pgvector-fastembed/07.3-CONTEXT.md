**Gathered:** 2026-03-08

<domain>
## Phase Boundary

Add persistent cross-run mission context with a 5-layer cascade retrieval system fused via Reciprocal Rank Fusion (RRF). Uses local ONNX embeddings only (fastembed, no paid APIs). Provides `MockEmbeddingProvider` for CI. Wires persistence into `ContextManager` so completed mission contexts survive process restarts and are shared across horizontal replicas. Adds a cross-run artifact store. Delivers a multi-replica stress test harness.

**What this phase does NOT do:**
- graph.py decomposition (→ Phase 7.4)
- LangGraph Send() parallel execution (→ Phase 7.5)
- LLM-generated summaries (post-7.3 stabilization)
- Kubernetes / cloud deploy (separate milestone)
- Cross-encoder reranking (post-7.3)

</domain>

<decisions>
## Implementation Decisions

### Embedding Stack
- Use `fastembed>=0.3` (Qdrant ONNX runtime), NOT sentence-transformers (no PyTorch)
- Model: `BAAI/bge-small-en-v1.5` — 384-dim, top MTEB score for its size class, ~24MB, CPU-native
- `MockEmbeddingProvider` returns deterministic 384-dim vectors via SHA-256 seeding — no model download in CI
- `EMBEDDING_PROVIDER` env var: `"fastembed"` | `"mock"` (default: `"mock"`)
- Add `fastembed>=0.3` as optional dep group `[context]` in pyproject.toml

### Vector Schema
- Use `vector(384)` throughout (migrate from `vector(1536)` in 002_foundation.sql if present)
- Binary quantization: `BIT(384)` column for Hamming-distance fast approx (L3)
- Full float32: `vector(384)` with HNSW index for cosine similarity (L4)
- HNSW not IVFFlat — better accuracy for small-to-medium collections (<1M rows)

### Retrieval Cascade (5 layers)
- L0: SHA-256 of normalized goal text → exact cache hit → short-circuit, skip L1-L4
- L1: 64-bit tool bitmask (`encode_tool_pattern`) → structural match → short-circuit, skip L2-L4
- L2: Postgres `tsvector` + `ts_rank` BM25 → keyword similarity (deterministic, native Postgres)
- L3: Binary Hamming distance on `BIT(384)` → fast semantic approx (L3 is computed inside Postgres as part of L4 query optimization — may be merged with L4 in implementation)
- L4: Float32 cosine via pgvector HNSW → full semantic search
- RRF fusion: Reciprocal Rank Fusion over L2 + L4 results (no tunable weights, proven robust)

### Database Tables
- `mission_contexts` table: `id, run_id, mission_id, goal, goal_hash, tool_pattern, goal_tsvector, embedding_bin, embedding, status, summary, tools_used, key_results, artifacts, created_at`
- `mission_artifacts` table: `id, run_id, mission_id, key, value, source_tool, key_hash, embedding, created_at`
- Migration 003: `mission_contexts` table + all indexes
- Migration 004: `mission_artifacts` table + indexes
- Migration 002 update: change `vector(1536)` → `vector(384)` if present

### ContextManager Integration
- `ContextManager.__init__` gains optional `mission_context_store` and `embedding_provider` params (both default None for backward compat)
- `on_mission_complete()` persists to DB via `_persist_mission_context()` (no-op if store is None)
- `build_planner_context_injection()` queries cascade for top-3 cross-run hits, formats as `[Cross-run] Similar: "{goal}" → {summary}`, bumps cap from 800 → 1500 chars
- All new params optional — zero backward-compat breaks

### graph.py Wiring
- `LangGraphOrchestrator.__init__` gains optional `embedding_provider` and `mission_context_store` params (both default None)
- Passes both to `ContextManager(...)` construction
- No other graph.py changes — decomposition is out of scope

### Tool Bitmask (L1)
- `TOOL_BITS` dict: sorted canonical tool list, each tool assigned a bit position
- New tools appended to preserve existing bitmasks
- `encode_tool_pattern(tools_used: list[str]) -> int`

### RRF Implementation
- `reciprocal_rank_fusion(*ranked_lists, k=60)` in `mission_context_store.py`
- Score = `sum(1 / (k + rank + 1))` across all input lists
- No tunable weights, proven +15-30% recall over single-list approaches

<specifics>
## Specific References

**Industry precedent:** Zep, Haystack, Hindsight MCP, Cognee all use deterministic+semantic cascade, never pure vector search.

**fastembed model download:** `BAAI/bge-small-en-v1.5` downloads ~24MB on first use, cached locally. CI uses MockEmbeddingProvider so no download in CI.

**Binary quantization recall:** 0.3% recall loss vs float32, 32x smaller storage, 10x faster search. Essentially free.

**pgvector BIT type syntax:** `BIT(384)` column, converted from float via sign bit: `"1" if v >= 0 else "0"`.

**HNSW index creation:** `CREATE INDEX ON mission_contexts USING hnsw (embedding vector_cosine_ops);`

**RRF k=60:** Standard parameter from Cormack et al. (2009), used unchanged in production systems.

**Existing storage layer:** See `src/agentic_workflows/storage/` — match connection/cursor patterns used by `PostgresCheckpointStore`, `PostgresMemoStore`, `PostgresRunStore`.

**Existing ContextManager:** `src/agentic_workflows/orchestration/langgraph/context_manager.py` — `large_result_threshold=1500`, `sliding_window_cap=20`, `build_planner_context_injection()` uses `policy_flags["injected_mission_ids"]` for dedup.
