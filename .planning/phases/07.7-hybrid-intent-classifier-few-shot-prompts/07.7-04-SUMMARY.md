## Decisions Made
- route_by_intent() accepts dict | None (not IntentClassification dataclass) because state stores intent_classification as a dict via to_dict() -- avoids import dependency and works with checkpoint serialization
- Multi-action output excluded from format drift detection: concatenated JSON objects naturally trigger parse fallback, which is expected behavior not format drift
- Escalation condition narrowed to: (single-action parse fallback) OR (validation fallback) -- prevents false positives on multi-action batches
- classifier_provider passed as self.provider from graph.py to parse_missions for LLM-backed intent classification
