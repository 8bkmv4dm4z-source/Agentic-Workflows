## Decisions Made

- Lazy import inside try block (not top-level): consistent with how MissionContextStore is imported in the same block — avoids psycopg_pool/ArtifactStore import failures in SQLite-only deployments
- Explicit `artifact_store = None` in except handler: even though it defaults to None, makes the failure-path behavior explicit alongside the `pool = None` reset
