## Implementation Details

### report_schema_compliance()

- Accepts `role`, `first_attempt_success`, optional `trace_id` and `run_id`
- Calls `client.create_score(name="schema_compliance", value=1.0|0.0, data_type="NUMERIC", comment="role=...")`
- Maps `run_id` to Langfuse `session_id` for cross-run grouping
- No-op when `get_langfuse_client()` returns None
- All exceptions silently suppressed via `contextlib.suppress(Exception)`

### graph.py Wiring

Three call sites in `_plan_next_action()`:
1. **Main parse path** (line ~1204) -- after `_parse_all_actions_json(model_output)`
2. **Cloud fallback path** (line ~1435) -- after cloud provider output parsing
3. **Context overflow retry path** (line ~1539) -- after compaction + retry parsing

Each reports `first_attempt_success=not fallback_used` with the active specialist role and run_id.
