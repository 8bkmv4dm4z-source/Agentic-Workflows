# Phase 7.6 Plan 01: Provider context_size() + Two-Tier Prompt System Summary

**One-liner:** Two-tier prompt system (compact/full) gated on provider.context_size() unblocks phi4/llama-cpp 8192-token models from context overflow.

## What Was Built

### Task 1: context_size() on ChatProvider Protocol and all 4 concrete classes + ScriptedProvider

Added `context_size() -> int` to:
- `ChatProvider` Protocol (stub `...`)
- `LlamaCppChatProvider`: reads `LLAMA_CPP_N_CTX` env var, defaults to 8192
- `GroqChatProvider`: hardcoded 32768
- `OpenAIChatProvider`: hardcoded 128000
- `OllamaChatProvider`: returns `self.num_ctx` if >0 else `OLLAMA_NUM_CTX` env (default 32768)
- `ScriptedProvider` (conftest.py): hardcoded 32768 — keeps all integration tests in full tier

### Task 2: _select_prompt_tier(), COMPACT directive, two-tier _build_system_prompt()

- `_select_prompt_tier(context_size: int) -> Literal["compact", "full"]`: ≤10000 → compact, >10000 → full
- `supervisor.md`: `## COMPACT` section prepended (5 lines, JSON-only directive)
- `LangGraphOrchestrator.__init__`: `self._prompt_tier = _select_prompt_tier(provider.context_size())` with AttributeError fallback for legacy test providers
- `_build_system_prompt()`: compact tier = env_block + COMPACT directive + tool names only; full tier = env_block prepended to existing prompt

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 2 - Missing Functionality] AttributeError fallback for legacy test providers**
- **Found during:** Task 2 test run
- **Issue:** Test providers like `_StrongProvider` in test_model_router_wiring.py do not implement `context_size()`. The new `__init__` call to `self.provider.context_size()` raised `AttributeError` for these, causing 28 test regressions.
- **Fix:** Wrapped `self.provider.context_size()` in try/except AttributeError; fallback to 32768 (full tier). This is the correct default — only new small-context providers opt into compact tier.
- **Files modified:** `src/agentic_workflows/orchestration/langgraph/graph.py`
