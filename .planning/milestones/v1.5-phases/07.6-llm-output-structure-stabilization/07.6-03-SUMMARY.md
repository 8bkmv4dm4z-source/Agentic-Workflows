Sub-task cursor persistence (E1-E4) implemented: chunked read_file_chunk progress now survives context eviction via upsert_cursor/get_cursor/get_active_cursors on MissionContextStore and cursor hint re-injection in ContextManager.compact().

## What Was Built

**MissionContextStore cursor methods (pool=None path):**
- `upsert_cursor(run_id, plan_step_id, mission_id, tool_name, key, cursor, total)` — stores in `self._cursors` dict with composite key
- `get_cursor(run_id, plan_step_id, mission_id, tool_name, key)` — returns `int | None`
- `get_active_cursors(run_id)` — returns list of cursor dicts filtered by run_id
- Postgres path: UPSERT into `sub_task_cursors` table (migration 005)

**ContextManager.compact() cursor hint injection:**
- After sliding window eviction, queries `get_active_cursors(run_id)`
- Appends one `[Orchestrator] Chunked read in progress: read_file_chunk path=<key> next_offset=<N> of <total> lines. Continue from offset=<N>.` message per active cursor
- Only fires when `self._store is not None` and `run_id` is non-empty

**graph.py executor result handler cursor tracking:**
- After `read_file_chunk` tool returns `has_more=True`: calls `_store.upsert_cursor()` with `next_offset` and `total_lines`
- After `has_more=False`: pops the cursor from `_store._cursors` (clear on completion)
- Uses `getattr(self, "context_manager", None)` to safely access the store

**graph.py __cursor_resume bypass:**
- Added `_is_cursor_resume` check before `seen_tool_signatures` duplicate detection
- Only bypasses when `action.get("__cursor_resume") is True AND tool_name == "read_file_chunk"`
- All other tools continue through normal dedup logic unchanged
