1. **src/agentic_workflows/api/routes/run.py** — Create APIRouter with prefix="" (mounted at root in app.py):

   **POST /run** (the core endpoint):
   - Accept RunRequest body and Request object (for client_ip).
   - Generate run_id (body.run_id or uuid4()).
   - Save run to store with status="running", user_input, prior_context, client_ip=request.client.host.
   - Create `anyio.create_memory_object_stream(max_buffer_size=100)` — send_stream and receive_stream.
   - Store receive_stream in `request.app.state.active_streams[run_id]` for GET /run/{id}/stream reconnection.
   - Define `async def producer(task_status=anyio.TASK_STATUS_IGNORED)`: inside, use `anyio.to_thread.run_sync` to call a sync function that:
     - Builds initial state via `new_run_state(orchestrator.system_prompt, body.user_input, run_id=run_id)`.
     - Handles prior_context merging (same logic as graph.py run() — copy the prior_context handling block).
     - Calls `ensure_state_defaults`, sets up missions via `parse_missions`, etc.
     - Calls `orchestrator._compiled.stream(state, config={"recursion_limit": orchestrator.max_steps * 9}, stream_mode="updates")`.
     - For each `(node_name, chunk)` yielded, push SSE events into send_stream using `anyio.from_thread.run_sync(send_stream.send, event)`.
     - Emit both UI tier (node_start/node_end) and debug tier (state_diff) events.
     - After stream exhausts, call `orchestrator._finalize(final_state)` to get audit report.
     - Save completed result to RunStore via update_run.
     - Emit make_run_complete event.
   - **Important alternative (simpler, recommended):** Instead of replicating all of run()'s setup logic, add a `run_streaming(user_input, run_id, *, prior_context, on_event: Callable)` method to LangGraphOrchestrator that wraps `_compiled.stream()` and calls `on_event(node_name, chunk)` on each transition, then calls `_finalize()`. This keeps audit logic intact. The sync `on_event` callback pushes events via `anyio.from_thread.run_sync`.
   - In the async producer, catch exceptions, emit make_error event, update run status to "failed".
   - Finally block: `await send_stream.aclose()`, remove from active_streams.
   - Define `async def event_generator()`: iterate `async for event in receive_stream: yield event`.
   - Return `EventSourceResponse(event_generator(), data_sender_callable=producer)` — OR if data_sender_callable isn't available in the installed version, use `asyncio.TaskGroup` to launch producer as background task and return `EventSourceResponse(event_generator())`.

   **GET /run/{run_id}**:
   - Read from RunStore via get_run(run_id).
   - If not found: return 404 ErrorResponse.
   - If status=="running": return RunStatusResponse with partial data (elapsed_s computed from created_at, missions_completed from stored progress, tools_used_so_far).
   - If status=="completed" or "failed": return RunStatusResponse with full result, audit_report, mission_reports.

   **GET /run/{run_id}/stream**:
   - Check if run_id exists in app.state.active_streams.
   - If not found: return 404 (run not in progress or already completed).
   - Session validation: verify request.client.host matches client_ip stored for this run_id. If mismatch, return 403.
   - Clone the receive_stream (or use the same one — since SSE is single-consumer, the original consumer must have disconnected for reconnection to make sense). Return EventSourceResponse from the stored receive_stream.
   - If the stream is already exhausted, return 410 Gone.

2. **src/agentic_workflows/api/app.py** — Update to include the run router. Add `from agentic_workflows.api.routes.run import router as run_router` and `app.include_router(run_router)`.

**Key design decision:** Use the simpler approach first — call `orchestrator.run()` via `run_in_threadpool` and emit SSE events by adding an `on_node_transition` callback parameter to `run()`. If modifying `run()` is too invasive, create a thin wrapper `run_streaming()` that calls `_compiled.stream()` directly and handles finalization. The research recommends the latter (Open Question 1 in RESEARCH.md).

**Anti-patterns to avoid:**
- Do NOT call `orchestrator.run()` synchronously inside an `async def` handler (blocks event loop).
- Do NOT create a new LangGraphOrchestrator per request (graph is compiled once at startup).
- Do NOT use `json.dumps(run_result)` without `default=str` (TypedDicts contain non-serializable types).
