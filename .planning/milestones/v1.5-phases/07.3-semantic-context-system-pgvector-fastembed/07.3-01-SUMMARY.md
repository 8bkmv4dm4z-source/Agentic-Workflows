## What Was Built

### Task 1: Updated 002_foundation.sql
Changed both `file_chunks.embedding` and `solved_tasks.embedding` from `vector(1536)` to `vector(384)` to match the FastEmbed BAAI/bge-small-en-v1.5 model's output dimension. Added a DO migration guard block that safely ALTERs pre-existing columns using `USING NULL::vector(384)` — safe because RESEARCH.md confirmed no code writes to these tables.

### Task 2: Created 003_mission_contexts.sql
Full DDL for the `mission_contexts` table supporting all four retrieval layers:
- **L0:** `goal_hash TEXT` + btree index for exact SHA-256 cache hits
- **L1:** `tool_pattern BIGINT` + btree index for 64-bit tool bitmask filtering
- **L2:** `goal_tsvector TSVECTOR` + GIN index + trigger for BM25 full-text search
- **L3:** `embedding_bin BIT(384)` for binary Hamming distance pre-filter
- **L4:** `embedding vector(384)` + HNSW index (cosine ops) for approximate nearest neighbor

Additional columns: `status`, `summary`, `tools_used TEXT[]`, `key_results JSONB`, `artifacts JSONB`, `created_at`.

Unique constraint on `(run_id, mission_id)` prevents duplicate context records.

### Task 2: Created 004_mission_artifacts.sql
DDL for the `mission_artifacts` table:
- `key_hash TEXT` + btree index for O(1) exact artifact lookup
- `embedding vector(384)` + HNSW index for semantic artifact search
- Unique constraint on `(run_id, mission_id, key)` for deduplication
