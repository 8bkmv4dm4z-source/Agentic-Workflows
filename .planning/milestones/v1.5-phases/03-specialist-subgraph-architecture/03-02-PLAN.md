Create the evaluator specialist module: an isolated EvaluatorState TypedDict and a
build_evaluator_subgraph() factory that compiles a single-node LangGraph StateGraph.
The subgraph accepts a state dict populated from mission run data, delegates to the
existing audit_run() function, and returns an updated EvaluatorState with eval_audit_report
populated. Unit tests verify isolation and correct delegation behavior.

Purpose: Phase 3's primary goal is testable isolation — each specialist must be a
self-contained compiled graph before Phase 4 wires them into the main graph.

IMPORTANT: audit_run() accepts kwargs named `missions`, `mission_reports`, and `tool_history`
(the RunState field names). The evaluate_node reads from EvaluatorState's prefixed fields
but must pass them to audit_run() using the correct kwarg names:
  - state["eval_missions"]        → missions=state["eval_missions"]
  - state["eval_mission_reports"] → mission_reports=state["eval_mission_reports"]
  - state["eval_tool_history"]    → tool_history=state["eval_tool_history"]

audit_run() does NOT accept eval_missions, eval_mission_reports, eval_tool_history,
eval_mission_contracts, or eval_audit_report as kwargs. Passing them raises TypeError.

Do NOT pass eval_mission_contracts to audit_run() at all — it is an EvaluatorState
documentation field only.

Stores the result as report.to_dict() in state["eval_audit_report"].
