    - What is a Dockerfile -- how it builds an image step by step
    - What is docker-compose -- defining multi-container systems declaratively

    **3. Postgres Persistence Architecture**
    - The three stores: CheckpointStore, RunStore, MemoStore
    - Protocol abstractions: why we defined CheckpointStore and MemoStore Protocols
    - SQLite vs Postgres implementations: same interface, different backends
    - The store factory pattern in app.py lifespan: how DATABASE_URL triggers the swap
    - SQL init scripts: what 001_init.sql and 002_foundation.sql create, why pgvector
    - Foundation tables (task_runs, file_chunks, solved_tasks): what they're for (v2 context management), why they're empty now

    **4. Sync vs Async Pool Decision**
    - Why CheckpointStore and MemoStore must be synchronous (called from sync graph nodes)
    - Why RunStore is async (called from FastAPI route handlers)
    - The single sync ConnectionPool approach: RunStore wraps sync calls with anyio.to_thread.run_sync
    - Why psycopg was chosen over asyncpg (per RESEARCH.md: LangGraph ecosystem uses psycopg, API incompatibility with AsyncPostgresSaver)
    - Connection pool configuration: autocommit=True (why), prepare_threshold=0 (why)

    **5. CI Pipeline Architecture**
    - The test matrix: SQLite leg vs Postgres leg
    - Postgres service container in GitHub Actions: how it works, health check options
    - Why init scripts run via psql in CI (no volume mount in GitHub Actions)
    - Docker build test job: build-only, no push
    - Coverage enforcement: pytest-cov with 80% threshold
    - ScriptedProvider: why no live LLM credentials in CI

    **6. Common Operations**
    - Starting the full stack: `docker compose up` (or `make docker-up`)
    - Starting just Postgres for local dev: `docker compose up postgres`
    - Resetting the database after schema changes: `docker compose down -v`
    - Running tests against Postgres locally: set DATABASE_URL, run pytest
    - Checking logs: `docker compose logs -f`

    **7. Known Constraints and Gotchas**
    - Docker init scripts only run on empty database (Pitfall 5 from RESEARCH.md)
    - prepare_threshold=0 required to avoid DuplicatePreparedStatement errors
    - psycopg[binary] bundles libpq -- no system deps needed
    - Postgres tests auto-skip without DATABASE_URL
