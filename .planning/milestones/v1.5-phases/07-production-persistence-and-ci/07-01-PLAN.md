<objective>
Create the Postgres persistence layer: Protocol abstractions for CheckpointStore and MemoStore, Postgres implementations of all three stores (CheckpointStore, MemoStore, RunStore), SQL init scripts, and ENV-based store factory wiring in the FastAPI lifespan.

Purpose: Replace SQLite with Postgres for production persistence while keeping SQLite for dev/test. This is the core PROD-03 requirement -- all data (checkpoints, runs, memos) moves to Postgres when DATABASE_URL is set.

Output: 5 new Python modules, 2 SQL migration scripts, modified app.py with store factory, updated pyproject.toml and .env.example.

**Driver deviation note:** CONTEXT.md locks "asyncpg" driver and "AsyncPostgresSaver" as drop-in replacement. RESEARCH.md (HIGH confidence) found that AsyncPostgresSaver's API (aput/aget/alist) is incompatible with the project's SQLiteCheckpointStore API (save/load_latest/list_checkpoints) -- NOT a drop-in replacement. Additionally, langgraph-checkpoint-postgres uses psycopg (not asyncpg). This plan uses psycopg[binary] + psycopg_pool with custom Postgres store implementations that mirror the SQLite APIs exactly, achieving the user's primary intent ("as little changes from P6 to P7") while using the correct, compatible driver. See RESEARCH.md "CRITICAL: Driver Mismatch Finding" section for full analysis.
</objective>
